Can be created easily by using the material schema for a table. The schema includes all necessary imports and creates a datasource class too. On the top there are also demo data entries, so the table does not look that empty.

ng g @angular/material --PATH-TO-COMPONENT--

Pros:
  • Very quick and effectiv for demonstrating the power of schemas
  • Imports Modules
  • Creates Testfiles with Module Imports
  • Adds Sorting
  • Adds Pagination
  • Datasource can be extended with ez
Cons:
  • Cell Templates can not be reused
  • For each table filtering must be recreated
  • Switch to virutal scrolling can not be configured
